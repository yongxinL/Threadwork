# /tw:entropy — Entropy Report Viewer

Display entropy reports generated by the tw-entropy-collector background agent.

## Usage

```
/tw:entropy                  Show latest entropy report for current phase
/tw:entropy history          List all entropy reports across all phases
/tw:entropy show <N>         Show entropy report for a specific wave number
```

## What This Shows

The entropy collector runs after each wave completes and scans for cross-output
inconsistencies: naming drift, import boundary violations, orphaned artifacts,
documentation staleness, inconsistent error handling, and duplicate logic.

## Instructions

When the user runs `/tw:entropy`:

1. Read `.threadwork/state/project.json` to get the current phase number.
2. List all files matching `entropy-report-wave-*.json` in `.threadwork/state/phases/phase-<N>/`.
3. Read the most recent report (highest wave number).
4. Format and display it.

When the user runs `/tw:entropy history`:

1. Find all entropy reports across all phases in `.threadwork/state/phases/`.
2. Display a summary table sorted by phase and wave.

When the user runs `/tw:entropy show <N>`:

1. Read the report for wave N in the current phase.
2. Display it with full issue details.

## Output Format

### Default (`/tw:entropy`)

```
── Entropy Report: Wave N, Phase M ─────────────────────
Generated: YYYY-MM-DD HH:MM UTC
Scanned files: X

Issues found: Y  |  Auto-fixed: Z  |  Queued for next wave: W

[If no issues]
✅ No entropy issues detected this wave.

[If issues exist — one line per issue]
⚠️  [type] file/path — description
    Severity: minor|warning|error | Auto-fix: applied|pending|not applicable
─────────────────────────────────────────────────────────
```

### History (`/tw:entropy history`)

```
── Entropy Report History ────────────────────────────────
Phase  Wave  Date        Issues  Auto-fixed  Queued
1      1     2025-02-14  3       1           2
1      2     2025-02-15  0       0           0
─────────────────────────────────────────────────────────
```

### Wave Detail (`/tw:entropy show <N>`)

Full JSON report reformatted as a readable list, with all issue details including
spec references and commit hashes for applied fixes.

## Skill-Tier Formatting

- **Ninja**: One-line summary only: `Wave N: X issues (Y fixed, Z queued)`
- **Advanced**: Summary + issue list without explanations
- **Beginner**: Summary + issue list + plain-English explanation of each issue type

## If No Reports Exist

```
No entropy reports found for phase <N>.
The tw-entropy-collector runs automatically after each wave completes.
If you've completed at least one wave, check that post-tool-use.js is configured
in your Claude Code settings.
```
